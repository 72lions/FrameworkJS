Introduction

FrameworkJS is an MVC framework with support for Models, Views, Controllers and Services. It supports a strip-down Publisher/Subscriber pattern and it also has a utility for doing AJAX calls.

The project is hosted on Github and you can report any issues on the Github issues page or you can tweet at @72lions

Dependencies

FrameworkJS is depended only on Douglas Crockford's JSON utility. The framework includes the library so you don't have to downloaded yourself.

FrameworkJS.Controller

A controller is responsible for talking to a view and a model and also passing data from the model to the view. A view cannot talk to a model and a model can't talk to a view.

Extending a controller

Extending a controller is really easy.


FrameworkJS.extend(function() {

    this.someFunction = function() {

        // Some code here...

    };

}, FrameworkJS.Controller, 'Controller.Grid');

The FrameworkJS.extend function takes three parameters. The first one is a function with the "class" structure, the second one is the base "class" and the last one is the "class" name. Since the "class" name is a string you can use something like "namespace.classname".

Instantiating a controller

var gridController = FrameworkJS.retrieve(
    'SomeId',
    'Controller.Grid');

FrameworkJS.retrieve takes three parameters. The first one is the unique id of the instance. The second parameter is the "class" name that we provided when we extended the base controller "class".

Get an instance of a controller

We do the exact same thing that we did when we wanted to instantiate a controller but this time we give the id of a controller that already exists. 

var alreadyInstantiatedGridController = FrameworkJS.retrieve(
    'SomeId',
    'Controller.Grid');
