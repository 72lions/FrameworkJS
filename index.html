Introduction

FrameworkJS is an MVC framework with support for Models, Views, Controllers and Services. It supports a strip-down Publisher/Subscriber pattern and it also has a utility for doing AJAX calls.

The project is hosted on Github and you can report any issues on the Github issues page or you can tweet at @72lions

Dependencies

FrameworkJS is depended only on Douglas Crockford's JSON utility. The framework includes the library so you don't have to downloaded yourself.

FrameworkJS.Controller

A controller is responsible for talking to a view and a model and also passing data from the model to the view. A view cannot talk to a model and a model can't talk to a view.

Private properites

_view       The controller's view. FrameworkJS.View

_model       The controller's model. FrameworkJS.Model

Public properties

name       The name of the controller.

id       The id of the controller.

Public methods

initialize       object.initialize([atrributes])
Responsible for initializing the controller. The attributes parameter is an object.

postInitialize       object.postInitialize()
This function is executed automatically after the initialize function of the object is executed. Any logic after the initialization should go here.

setView       object.setView(view)
Sets the view of the controller. The view parameter should be a FrameworkJS.View or an object that extends the FrameworkJS.View object.

getView       object.getView()
Returns the view of the controller. The view that is returned is a FrameworkJS.View or an object that extends the FrameworkJS.View object.

setModel       object.setModel(model)
Sets the model of the controller. The model parameter should be a FrameworkJS.Model or an object that extends the FrameworkJS.Model object.

getModel       object.getModel()
Returns the model of the controller. The model that is returned is a FrameworkJS.Model or an object that extends the FrameworkJS.Model object.

onModelChange       object.onModelChange(event)
Triggered when a property of a model is changed. The event is an object with two keys: type and value.

onNotify       object.onNotify(arguments)
Triggered when a message that the controller is listening to is published by the FrameworkJS.Publisher. The arguments parameter is an object with two keys: message and params.

subscribe       object.subscribe(message)
Subscribes to a specific message. The message parameter is a string.

unsubscribe       object.unsubscribe(message)
Unsubscribes from a specific message. The message parameter is a string.

retrieve       object.retrieve(id, classType)
Retrieves an instance of a class. The id parameter is a string and it's the id of the class that we want to retrieve. The classType parameter is a string and its the class type of the instance that we need. If an instance with that type and id exists then that one will be returned, else it will insantiate a new one.

Extending a controller

The FrameworkJS.extend function takes three parameters. The first one is a function with the "class" structure, the second one is the base "class" and the last one is the "class" name. Since the "class" name is a string you can use something like "namespace.classname".

Instantiating a controller

FrameworkJS.retrieve takes three parameters. The first one is the unique id of the instance. The second parameter is the "class" name that we provided when we extended the base controller "class".

FrameworkJS.View

A view is responsible for manipulating the DOM and listening for DOM events. Any logic that has to do with DOM manipulation should exist inside a view. If a view needs to talk to a model or another controller or view then this has to happen through the view's controller.

Private properties

_controller       The controller that the view belongs to. FrameworkJS.Controller

_domElement       The view's main DOM element.

Public properties

name       The name of the view.

id       The id of the view.

title       The title of the view.

Public methods

initialize       object.initialize([atrributes])
Responsible for initializing the view. The attributes parameter is an object.

show       object.show()
May be used for showing the view.

hide       object.hide()
May be used for hiding the view.

getModel       object.getModel()
Returns the model of the view's controller by talking to the controller. The model that is returned is a FrameworkJS.Model or an object that extends the FrameworkJS.Model object.

setController       object.setController(controller)
Sets the view's controller. The controller paramater is a FrameworkJS.Controller or an object that extends the FrameworkJS.Controller object.

getController       object.getController()
Returns the view's controller. The controller that is returned is a FrameworkJS.Controller or an object that extends the FrameworkJS.Controller object.

getDOMElement       object.getDOMElement()
Returns the view's DOM Element.

initialize       object.initialize()
Initializes the view

FrameworkJS.Model

A model is responsible for storing data.

Private properites

_data       The object that holds the data.

Public properties

name       The name of the model.

id       The id of the model.

Public methods

set       object.set(properties, silent)
Setting the model's data. The properties parameter is an Object with keys/values. The silent parameter is a Boolean and if it is true then no change event is dispatched when a property of the model changes.

get       object.get(key)
Returns a property value of the model. The key parameter is a String.
